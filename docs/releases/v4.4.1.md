# v4.4.1 Release Notes

`v4.4.1` is a patch release for calendar-day difference semantics in display-oriented date comparisons.

## Fixed

- Added `differenceInCalendarDays()` for local date-boundary counts where elapsed 24-hour duration math is not the right model.
- Updated `formatCalendarDate()` to use calendar-day distance, preventing near-future date-only values from drifting into an extra displayed day.
- Added regression coverage for the May 22 to May 24 case.

## Verification

- `npm run release:verify`
