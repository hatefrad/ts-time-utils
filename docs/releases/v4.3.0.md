# v4.3.0 Release Notes

## Summary

`v4.3.0` is the correctness release. It focuses on deterministic hard-edge coverage, timezone overlap behavior, and cross-module consumer realism.

## Highlights

- Timezone and DST regression coverage now includes deterministic northern- and southern-hemisphere fixtures.
- `findCommonWorkingHours()` now returns the longest contiguous overlap window consistently, including wrapped and full-day cases.
- Consumer-style integration tests now cover built-package imports, recurrence-to-format flows, and parse/validate/calculate interoperability.
- The public type surface was tightened where docs had implied stronger guarantees than the implementation provided.

## Verification

- `npm run lint`
- `npm test`
- `npm run build`
- `npm run test:package`

## Notes

- Working-hours helpers remain local-clock utilities; timezone adaptation should happen through the timezone module before those helpers are used.
