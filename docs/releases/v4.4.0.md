# v4.4.0 Release Notes

## Summary

`v4.4.0` is the first public release after the roadmap cleanup. It rolls the trust, correctness, and productization slices into one npm release while refreshing adoption docs and standardizing the maintainer release path.

## Highlights

- Natural-language and working-hours public contracts now match implemented behavior.
- Plugin registration and built-package verification are now explicit and tested.
- Timezone overlap handling and cross-module integration coverage were hardened with deterministic tests.
- README, framework guides, and migration docs now point at exported APIs only.
- Module-selection guidance and a brief “when not to use this library” section were added to the README.
- CI and publish workflows now share the same release-readiness command: `npm run release:verify`.
- Versioned release notes now live under `docs/releases/`, with `v4.2.0` and `v4.3.0` preserved as roadmap milestone notes.

## Release Workflow

1. Update `CHANGELOG.md`.
2. Update the matching `docs/releases/vX.Y.Z.md` note.
3. Run `npm run release:verify`.
4. Tag the release as `vX.Y.Z` and push the tag.

## Verification

- `npm run release:verify`
